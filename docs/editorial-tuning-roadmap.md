# Editorial Tuning Roadmap v0.1

The system has proven it can move from weekly planning to episode brief to prompt pack to a first script draft. Future improvements should focus primarily on voice fidelity, scripture handling, and editorial quality rather than additional architecture.

## Phase 1: Scripture Enrichment

Proverbs Daily script generation should include richer scripture handling before draft generation.

### Translation Strategy

Primary translation:

- NIV

Secondary translations:

- NKJV
- MSG
- NLT
- ESV

### Guidelines

- NIV is the default public-facing translation.
- NKJV may be used when poetic structure matters.
- MSG may be used when modern-language clarity helps illuminate the proverb.
- NLT may be used when readability is important.
- ESV may be used for precision or word-study moments.
- Future script generation should include verse text in the episode brief.
- Scripture remains the star of the episode.

## Phase 2: Fred Texture Layer

Future scripts should include lived texture when naturally relevant. This should not be forced into every episode, but the engine should have access to recurring life domains that help the script sound more like Fred.

Recurring life domains:

- Family
- Marriage
- Fatherhood
- Chaplaincy
- Baylor Hospital experiences
- ACPE/CPE training
- Music and creativity
- P.I.D.
- Sacred Hum
- Proverbs Daily development
- Technology and AI
- Fitness, sleep, and wellbeing
- Everyday observations

The purpose of this layer is not autobiography for its own sake. It is to give wisdom a lived room to stand in.

## Phase 3: Style Library

Future versions should build a small library of style examples by episode category.

Future category examples:

- Family episodes
- Technology episodes
- Music episodes
- Chaplaincy episodes
- Humor episodes
- Deep theological episodes

The style library should preserve and clarify:

- strong hooks
- memorable Remember statements
- practical challenges
- conversational cadence

This library should help the Script Engine produce drafts that feel less generic and more recognizably Proverbs Daily.

## Phase 4: Script Validation

Future script generation should include automated or semi-automated validation checks.

Validation checks:

- runtime target
- hard runtime maximum
- prayer length
- challenge inclusion
- Reader Version completeness
- Performance Version completeness
- ElevenLabs safety

Validation should support editorial review, not replace it. A script can pass structural checks and still need Fred's ear, memory, humor, or pastoral judgment.
